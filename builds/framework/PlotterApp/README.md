## System design

### Layer architecture

```
┌─────────────────────────────────────────────────────────────┐
│  Compose Desktop Client (Kotlin) — :app / :core / :cli      │
│  • Blueprint editor    • Flows editor    • Project generator │
│  • Debugger panel      • Test runner                        │
└───────────────────────┬─────────────────────────────────────┘
                        │ generate
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  Generated Project Tree — builds/framework/PlotterApp/   │
│  • blueprint.json  • flows.json  • src/  • python/  • ui/   │
│  • scripts/  • shared/  • zig-services/  • assets/           │
└───────────────────────┬─────────────────────────────────────┘
                        │ zig build
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  Native Binary — SDL3 + ImGui + sol2 + pybind11 + Zig C ABI │
│  • C++20 MVC loop    • Lua 5.4 panels    • Python analytics  │
│  • TS/XHTML UI       • FlowRunner        • NexusBridge       │
└─────────────────────────────────────────────────────────────┘
```

All layers communicate in-process. The generation boundary (Kotlin → native sources) is crossed at project creation; the build boundary (Zig → binary) at compile time; the runtime boundary (sol2, pybind11, C ABI) every frame.

### Data flow

```
User input (mouse/keyboard/touch)
    │
    ▼
SDL3 event loop → ImGui_ImplSDL3_ProcessEvent()
    │
    ▼
ImGui::NewFrame() → AppView::draw() → LuaPanels::drawFrame()
    │                                            │
    ▼                                            ▼
AppController::onCommand()              Lua script callbacks
    │                                            │
    ├── PythonEngine::call("fn", args)           └── nxs.bridge.call_cpp("cmd")
    ├── AppModel::mutate()
    └── FlowRunner::tick()
```

## Memory & thread model

### Threading

| Thread         | Role                                               | Stack   |
|---------------|---------------------------------------------------|---------|
| **Main**       | SDL3 event loop, ImGui render, all C++ MVC         | Default |
| **Python**     | pybind11 holds the GIL; Python calls release GIL   | 8 MB    |
| **FlowRunner** | Background flow ticks on main thread (`tick()`)    | N/A     |
| **Zig alloc**  | C ABI calls from any thread; arena is NOT thread-safe | N/A   |

There is exactly **one thread** for the MVC loop. Python releases the GIL during long numpy operations. FlowRunner ticks are synchronous and must complete under 16ms to avoid frame drops. For heavy async work, add a C++ worker thread and communicate via `std::atomic` or a lock-free queue (not provided by default — add as needed).

### Memory allocation

| Layer       | Strategy                                      | Notes                                  |
|------------|-----------------------------------------------|----------------------------------------|
| C++ model   | RAII — `std::vector`, `std::string`, `std::unique_ptr` | All `[[nodiscard]]`, `constexpr` where possible |
| C++ view    | Stack-allocated draw data per frame            | Cleared each `ImGui::NewFrame()`        |
| Python      | pybind11 buffer protocol — shares numpy memory | No copy across C++/Python boundary     |
| Lua         | sol2 userdata — pointers to C++ objects        | Lifetime managed by C++                 |
| Zig         | ArenaAllocator (opt-in via `nxs_config.json`)  | Reset per frame in `PlotController::refresh` |

### RAII ownership rules

```
AppModel (singleton, constructed in main)
    │
    ├── FunctionRegistry ── owns ──> vector<Function> (function definitions)
    │
AppController (references model)
    │
    ├── PythonEngine ── opens ──> CPython interpreter (startup)
    │
AppView (references controller)
    │
    ├── LuaPanels ── opens ──> sol2::state (startup)
    │
FlowRunner (references controller)
    │
    └── FlowDefinition[] ── loaded from ──> flows.json
```

No global state. No singletons (except the SDL/ImGui globals). All ownership is linear and explicit.

## Cross-language interop

### Calling convention matrix

| From ↓    | To → C++                      | To → Python                     | To → Lua                      | To → Zig                     |
|-----------|------------------------------|---------------------------------|------------------------------|-----------------------------|
| **C++**    | Direct function call          | `PythonEngine::call("fn", arg)` | `LuaPanels::call("fn", arg)` | `nxs_alloc(sz)` / `nxs_free(p)` |
| **Python** | `call_cpp("counter.get", "")` | Direct Python eval              | Via C++ bridge               | Via C++ bridge              |
| **Lua**    | `nxs.bridge.call_cpp(...)`    | `nxs.bridge.call_python(...)`   | `nxs.bridge.call_lua(...)`   | `nxs.bridge.call_zig(...)`  |
| **Zig**    | C ABI: `c.nxs_bridge_call()` | Via C++ bridge                 | Via C++ bridge               | Direct Zig fn               |

### NexusBridge registry

The `bridge.cppm` module implements a string-keyed function registry. Any language can register and any language can call:

```cpp
// C++: register
nxs::bridge::registry::set("counter.get", [&](std::string_view) {
    return std::to_string(m_model.counter());
});

// Python: call
call_cpp("counter.get", "")  // returns "0"

// Lua: call
nxs.bridge.call_cpp("counter.get", "")  // returns "0"
```

### Python → C++ data flow (pybind11)

```
Python: np.linspace(0, 1, 1000) → numpy array
    │
    ▼
pybind11 buffer protocol → pointer to double[] in numpy's memory
    │
    ▼
C++: std::span<const double> — zero copy, same address space
```

### Lua → C++ data flow (sol2)

```
Lua: ui.button("Click") → callback → nxs.bridge.call_cpp("cmd")
    │
    ▼
sol2: Lua function invokes C++ lambda via sol::protected_function
    │
    ▼
C++: AppController::onCommand("cmd") — same stack frame
```

## Build system

### Architecture (zig-services/build.zig)

```
build.zig
    │
    ├── module_lib (static library) — all .cppm module interface units compiled as C++20 modules
    │   ├── app_module_sources (9 files — AppModel, AppController, etc.)
    │   ├── shared_module_iface_sources (7 files — font_config, theme, etc.)
    │   ├── shared_module_impl_sources (4 files — .cpp implementations)
    │   └── shared_sources (ScriptArchive.cpp, ZigAllocator.cpp, etc.)
    │
    ├── exe — PlotterApp (main executable)
    │   ├── main.cppm — imports all modules
    │   ├── linkLibrary(module_lib)
    │   ├── linkLibrary(imgui_bundle)
    │   ├── linkLibrary(nexus_zig)  ← Zig C ABI exports
    │   ├── linkLibrary(sdl3)
    │   ├── linkSystemLibrary("GL")
    │   └── linkSystemLibrary("python3.12")
    │
    ├── pack_archive — host tool for lua.dat / python.dat
    ├── smoke_test — C++ ↔ Zig C ABI link verification
    └── nexus_zig — Zig library (allocator, C ABI exports)
```

### Module compilation pipeline

```
.cppm file
    │
    ├── clang -std=c++20 -fmodules-ts -fmodules-codegen -fmodules-debuginfo
    │
    ├── → .pcm (precompiled module) — consumed by importers
    └── → .o  (object file) — linked into module_lib
```

## Implementation guide

### Adding C++ code

1. Create `.cppm` file in `src/model/`, `src/controller/`, or `src/view/`
2. Add to `app_module_sources` in `zig-services/build.zig`
3. Declare exports with `export module nxs.desktop.<layer>;`
4. Rebuild: `cd zig-services && zig build`

Example new model:

```cpp
// src/model/TradingModel.cppm
export module nxs.desktop.trading;
import nxs.desktop.model;

export class TradingModel {
public:
    auto positions() const -> const std::vector<Position>& { return m_positions; }
    void addPosition(Position p) { m_positions.push_back(std::move(p)); }
private:
    std::vector<Position> m_positions;
};
```

### Adding Python analytics

```python
# python/functions.py
import numpy as np

def evaluate(func: str, x_min: float, x_max: float, samples: int) -> tuple:
    xs = np.linspace(x_min, x_max, samples)
    ys = eval(func)(xs)
    return xs, ys  # zero-copy via pybind11 buffer protocol
```

Install Python deps: `pip install -r requirements.txt`

### Adding Lua panels

```lua
-- scripts/panels.lua
nxs.register_panel("Analytics", function()
    if ui.button("Refresh") then
        nxs.bridge.call_cpp("analytics.refresh", "")
    end
    ui.same_line()
    ui.text("Last run: " .. nxs.bridge.call_cpp("analytics.lastRun", ""))
end)

nxs.register_hotkey(keys.F5, function()
    nxs.bridge.call_cpp("analytics.refresh", "")
end)
```

Lua hot-reloads at runtime — save `panels.lua` and the app picks up changes on next panel draw.

### Adding TypeScript/XHTML UI

```xml
<!-- ui/ui.xhtml -->
<panel id="sidebar" title="Controls" width="300">
    <slider bind="sampleCount" min="100" max="10000" label="Samples"/>
    <combo id="function-picker" items-source="availableFunctions" bind="pendingFunction"/>
    <button label="Add Curve" on-click="addPending"/>
    <plot id="main-plot" width="600" height="400" title="Waveform"/>
</panel>
```

```typescript
// ui/ui.ts
class AppPage {
    sampleCount = state<number>(1000);
    availableFunctions = state<string[]>(["sin(x)", "cos(x)", "x^2"]);
    pendingFunction = state<string>("sin(x)");

    addPending(): void {
```

The TS/XHTML toolchain lowers these to Lua/ImGui calls at build time.

## Blueprint & flows

### blueprint.json schema

The app graph uses five node types connected by directed edges:

| Node type        | Generated artifact                      | Runtime role                     |
|-----------------|----------------------------------------|----------------------------------|
| `python.module`  | `python/functions.py`                   | In-process NumPy analytics       |
| `cpp.model`      | `src/model/`                             | C++20 domain state with RAII     |
| `cpp.controller` | `src/controller/`                        | Commands, event wiring           |
| `ui.page`        | `ui/ui.xhtml` + `ui/ui.ts`              | Declarative page layout          |
| `lua.script`     | `scripts/panels.lua`                     | Live-editable panels and hotkeys |

Edges define data flow: `evaluate` → `sampleCache` → `activeCurves` → `commands`.

See [blueprint-schema.md](../../docs/templates/blueprint-schema.md) for full schema reference.

### flows.json schema

| Mode         | Trigger               | Use case                                    |
|-------------|-----------------------|--------------------------------------------|
| `background` | `interval` every N ms | Poll sensor, check queue depth              |
| `triggered`  | `event` + condition   | React to data arrival, connection state     |
| `startup`    | App launch            | Preload datasets, init hardware             |
| `manual`     | User action           | On-demand analysis pipeline                 |

### Langflow import

Design AI flows in [Langflow](https://github.com/langflow-ai/langflow) (visual DAG editor), export as JSON, and import via the Nexus client's Flows Editor → **Import Langflow JSON**. Safety default: every imported flow starts with `enabled: false`.

Component mapping:

| Langflow component | Nexus flow step     |
|-------------------|---------------------|
| `ChatInput`       | `manual` trigger    |
| `Schedule`        | `interval` trigger  |
| `Webhook`         | `event` trigger     |
| `Agent` / `LLM`   | `invoke` steps      |

## Style guide

### C++20

| Rule                    | Enforcement                           |
|-------------------------|---------------------------------------|
| RAII for all resources  | No raw `new`/`delete` without comment |
| `std::span` over raw pointers | `const char*, size_t` → `std::string_view` |
| `[[nodiscard]]`         | Every getter, query, factory          |
| `constexpr`             | Compile-time constants                |
| No C-style casts        | `static_cast<T>()` with rationale     |
| Brace initialization    | `int x{0}` — prevents narrowing       |
| `.cppm` for modules     | 22 interface units across templates   |
| `.clang-format`         | LLVM-derived, 4-space indent, 110 cols |

### Zig

- `zig fmt` before commits
- Explicit allocators everywhere
- Error unions (`!T`) — propagate with `try`/`catch`
- `callconv(.C)` on function pointers consumed from C++

### Lua

| Rule                        | Reason                                    |
|-----------------------------|-------------------------------------------|
| 2-space indent              | Consistency with sol2 examples            |
| snake_case functions/locals | Lua convention                            |
| Side effects in controller  | Keep panel layout logic pure              |
| Short header comment        | Document blueprint edge dependencies      |

### TypeScript

| Rule                               | Reason                                  |
|------------------------------------|-----------------------------------------|
| Semicolons                         | Consistent style                        |
| 4-space indent                     | Match surrounding code                   |
| `state<T>()` / `native<T>()`       | DSL binding types                       |
| `readonly` on non-mutated fields   | Compile-time safety                     |
| Page class name matches blueprint  | `blueprint.json` `ui.page` refs must match |

### Python

| Rule                       | Reason                               |
|----------------------------|--------------------------------------|
| PEP 8, 4-space indent      | Standard Python                      |
| Type hints on public fns   | C++ callers use the signatures       |
| numpy tuples for C++       | pybind11 buffer protocol expects them |

### Common issues

| Symptom                         | Likely cause                             | Fix                                           |
|--------------------------------|-----------------------------------------|----------------------------------------------|
| `SDL_Init failed`               | Missing SDL3 runtime                     | `apt install libsdl3-dev`                     |
| `Python import error`           | Missing Python package                   | `pip install -r requirements.txt`             |
| `Module 'nxs.*' not found`      | `.cppm` path missing in `build.zig`      | Add to `app_module_sources`                   |
| `Lua panel not showing`         | Syntax error in `panels.lua`             | Check stdout for Lua errors                   |
| `undefined symbol: _ZN3nxs...` | `.cpp` not registered in build           | Add to `module_lib` or `exe` sources          |
| `Flows not running`             | `flows.enabled: false` in config         | Set `"flows": { "enabled": true }`            |
| `Segfault in Lua callback`      | Lua userdata lifetime exceeded           | Ensure C++ object outlives Lua reference      |
