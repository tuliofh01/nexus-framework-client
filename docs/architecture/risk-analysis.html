Critical & high risks

Critical C1 — Docs promise wizard; code is counter demo

App.kt launches CounterScreen only. No ProjectGenerator, wizard, or imnodes editor. Diagrams and docs/architecture/overview.md still claim full wizard.

Critical C2 — Dual-repo template drift

This repo: template/desktop-app/. Client repo: templates/basic-app/. nxs_config.json references nexus-framework-client. Manual sync — no CI gate.

High H1 — JDK 21 (README) vs 26 (buildSrc) vs 17 (Android)

Three JDK targets documented inconsistently. Contributors install wrong toolchain.

High H2 — FetchContent network SPOF

7 Git repos at CMake configure (~50–100 MB). GitHub outage or air-gap = build failure. No SHA256 pin.

High H3 — Android Kotlin duplicates

MainActivity.kt and NexusApplication.kt contain duplicate class definitions — template does not compile.

High H4 — Android boot chain fragility

Chaquopy → Djinni bridge → SDL_main. Any step failure → empty charts or JNI crash.

High H5 — Python on UI thread

controller.refresh() every frame; sync numpy eval (64–4096 samples). 5 curves × 4096 → <10 FPS.
