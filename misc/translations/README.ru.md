## Что такое Nexus?

**Nexus Framework** — это **генератор нативных приложений** — и мы имеем в виду *настоящий натив*, не «мы обернули браузер и назвали нативным».

- Вы проектируете структуру приложения в визуальном графе (называемом **blueprint**)
- Nexus *генерирует* полностью готовый к продакшену нативный проект из шаблонов
- Вы компилируете в маленький бинарник (3-20МБ) который запускается за menos 200мс
- **Без браузера. Без JavaScript. Без «когда npm стал религией?»**

### Для кого это?

| Сценарий | Идеально? | Почему |
|:---|:---|:---|
| IoT планшеты в поле | **Абсолютно** | Нативная производительность, крошечный размер, оффлайн |
| Научная визуализация | **Абсолютно** | Графики в реальном времени, интеграция ImPlot |
| Промышленные UI | **Абсолютно** | SDL3 + ImGui = отзывчивые, детерминированные UI |
| Встраиваемые инструменты | **Абсолютно** | Zig cross-компиляция, нативные C++ тулчейны |
| Десктоп утилиты | **Да** | Маленькие бинарники, быстрый старт, без мусора |
| Android ruggedized | **Да** | Zig JNI, Chaquopy, работает на Android 8.0+ |
| AI/ML дашборды | **Да** | Интеграция Python, обработка данных в реальном времени |
| iOS приложения | **Пока нет** | Мы работаем над этим — терпение, падаван |
| Маркетинговые сайты | **Нет** | Используйте React Native или Flutter |
| Чистые Python приложения | **Не идеально** | Если только вы не любите смотреть как ваш скрипт компилируется 45 минут |

### Ментальная модель

Представьте что вы строите приложение как дом:

1. **Архитектор** (клиент Compose Desktop) — Вы рисуете blueprint
2. **Строительная бригада** (ProjectGenerator) — Nexus превращает в реальность
3. **Стройматериалы** (шаблоны C++/Lua/Python) — Готовы к использованию
4. **Готовый дом** (ваш нативный бинарник) — Вы красите стены и въезжаете

**Магия:** Вам не нужно понимать CMake, Zig или как работает SDL3. Nexus абстрагирует всё это.

## Быстрый Старт

```bash
# 1. Bootstrap (устанавливает JDK 26 + Zig 0.16.0):
zig run misc/client-setup/setup.zig && source misc/client-setup/env.sh

# 2. Компиляция генератора:
./misc/build_client.sh

# 3. Генерация десктоп приложения:
./gradlew :cli:run --args="generate --type desktop --name MyApp"

# 4. Компиляция бинарника:
cd builds/framework/MyApp && ./build_app.sh

# 5. Запуск:
./MyApp
```

**Результат:** Бинарник 3-20МБ запускается за 172мс, использует 42МБ RAM, работает оффлайн.

## Шаблоны: Десктоп vs Android

### Десктоп (SDL3 + ImGui + C++20+)

| Компонент | Что делает | Почему важно |
|:---|:---|:---|
| **SDL3** | Окна, GPU, ввод | Кроссплатформенность которая реально работает |
| **Dear ImGui** | Нативный immediate-mode UI | Та же библиотека что в Unity Editor |
| **Lua 5.4** | Скрипты, горячая перезагрузка | 200 строк заменяют 2000 строк C++ |
| **Python 3** | AI/ML, аналитика | pybind11 = скорость C++ с эргономикой Python |
| **Zig** | Build-система, кроссплатформенность | Одна команда компилирует для Linux, macOS, Windows |

### Android (Zig JNI + Chaquopy)

| Компонент | Что делает | Почему важно |
|:---|:---|:---|
| **SDL3 GLES** | Плавная графика на мобайле | 60fps гарантировано |
| **ImGui + Нативные виджеты** | Гибридный UI | Скорость ImGui + знакомство Android |
| **Lua 5.4** | Тот же скриптовый движок | Пишешь один раз — работаешь везде |
| **Chaquopy** | Управляемый Python runtime | Ответ Android на Python |
| **Zig JNI** | Мост Zig  Android Java | Без boilerplate Djinni кода |

| Метрика | Десктоп | Android | Эквивалент Electron |
|:---|:---|:---|:---|
| Размер бинарника | 3-20МБ | 5-25МБ | 100-500МБ+ |
| Время запуска | 170мс | 350мс | 1200мс+ |
| Использование RAM | 10-50МБ | 30-100МБ | 200-500МБ+ |
| Оффлайн? | Да | Да | Часто требует кэш |

## Стек Языков

| Язык | Роль | Почему именно этот? |
|:---|:---|:---|
| **C++20** | Горячий путь, модель, общий runtime | Zero-cost абстракции, SDL3/ImGui нативно |
| **Lua 5.4** | Скриптуемые панели, горячая логика | Встраиваемый, быстрый, 200 строк заменяют 2000 |
| **Python 3** | AI/ML, аналитика, data science | pybind11 = скорость C++ с эргономикой Python |
| **TypeScript/XHTML** | DSL для UI | Знаете HTML/CSS? Вы уже знаете Nexus UI |
| **Zig 0.16.0** | Sidecars, аллокатор, JNI bridge | Нативный C ABI, без зависимости от libc |

### Почему C++20, а не Rust?

Оба отличные. Nexus выбрал C++ потому что вся экосистема с которой он интегрируется уже была на C++. Это не *лучше* — это то что *работало*.

## Flows и UI

**Flows** — это автоматизации которые работают *внутри* приложения. Не на сервере. Не в облаке. Без ежемесячной оплаты.

- Визуальный дизайн через **Langflow** → экспорт JSON → импорт через CLI → запуск автоматически
- **Dear ImGui** для нативного UI (та же библиотека что в Unity Editor)
- **Lua** для скриптуемых виджетов
- **Python + Lua** для AI/ML + UI в реальном времени

| Управление памятью | Ручной аллокатор | ZigAllocator | Без утечек |

## Финальный Вердикт

**Да, если:**
- Вы хотите **крошечные** приложения (3-20МБ)
- Вам важна **скорость** (<200мс запуск)
- Вы ненавидите **раздутие памяти** (десятки vs сотни МБ)
- Вам нужна **оффлайн-способность**
- Вы разрабатываете для **десктопа или Android**

**Нет, если:**
- Вам нужна **поддержка iOS** (мы работаем над этим)
- Вы хотите создавать **маркетинговые сайты** (используйте React Native или Flutter)
- Вы довольны **JavaScript экосистемами**

**Nexus не для всех — и это нормально.** Мы строим что-то другое: **генератор нативных приложений** для людей которым важны **производительность, размер и эффективность.**

*Идите и создайте что-нибудь потрясающее.*
