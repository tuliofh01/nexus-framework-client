# {{projectName}} — Nexus Android App

Generated by [The Nexus Framework](https://github.com/tuliofh01/nexus-framework-client) — a native app scaffold that turns visual blueprints into compiled C++/Lua/Python binaries. SDL3 GLES + ImGui on Android, Chaquopy Python, Zig JNI bridge.

## System structure

```
{{projectName}}/
├── app/                    # Android application module
│   ├── build.gradle.kts    # Gradle build — APK + Chaquopy + Zig JNI
│   └── src/main/
│       ├── AndroidManifest.xml
│       ├── java/com/nexus/{{packageName}}/
│       │   ├── MainActivity.kt      # SDL activity + Python init
│       │   ├── AppCore.kt           # JNI bridge host
│       │   └── PythonBridge.kt      # Chaquopy interface
│       ├── python/                  # Python sources (Chaquopy)
│       └── res/                     # Android resources
├── src/                    # C++20 MVC
│   ├── main.cpp            # SDL_main — same loop as desktop
│   ├── model/              # Domain state
│   ├── controller/         # Commands + Python bridge
│   └── view/               # ImGui screens + Lua panels
├── shared/                 # Cross-target runtime helpers
├── scripts/                # Lua panels
├── zig-services/           # Zig JNI bridge
│   ├── build.zig           # Cross-compile .so
│   └── jni/                # Hand-authored JNI entry points
├── blueprint.json          # App structure graph
├── flows/                  # Runtime automations
└── nxs_config.json         # Project config
```

### Boot sequence

1. `AppCore` loads native `.so` via `System.loadLibrary`
2. `MainActivity.onCreate` calls `AppCore.installPythonBridge(PythonBridge)`
3. SDL3 runs `SDL_main` in `src/main.cpp` — same MVC loop as desktop

## Quick start

```bash
# 1. Compile native .so for device
cd zig-services
zig build -Dtarget=aarch64-linux-android

# 2. Build full APK
cd ..
./gradlew :app:assembleDebug
```

Requires: Android SDK, NDK r26+ (API ≥ 29), Zig 0.14.x, JDK 17+.

## Implementation guide

### Adding C++ code

Works the same as desktop — edit `src/model/`, `src/controller/`, `src/view/` and add source paths to `zig-services/build.zig`. Rebuild `.so` with `zig build -Dtarget=aarch64-linux-android`.

### Adding Python (Chaquopy)

Edit `app/src/main/python/helpers.py`:

```python
def greeting(project_name: str) -> str:
    return f"Hello from Python, {project_name}!"
```

Chaquopy bundling is automatic via Gradle. Add pip deps in `app/build.gradle.kts`:

```kotlin
chaquopy {
    defaultConfig {
        pip { install("numpy") }
    }
}
```

### Adding Lua panels

Edit `scripts/panels.lua` — identical to desktop. Lua packs to `assets/lua.dat` via Gradle `packLuaDat` task.

### Adding JNI bridge methods

The bridge lives in `zig-services/jni/`:

| File                     | Role                                                  |
|-------------------------|------------------------------------------------------|
| `python_bridge.zig`      | Pure-Zig JNI bridge — stores `JavaVM*`, bridge ref,  |
|                          | method IDs; exports 5 C ABI functions                |
| `lua_bridge.zig`         | Lua bridge JNI callbacks (optional, same pattern)    |

The bridge was rewritten from 7 C++ files (`jni_bridge.cpp`, `app_core.cpp/.hpp`, `NativePythonBridge.cpp/.hpp`, `python_bridge.hpp`, `eval_result.hpp`) to a single pure-Zig file. C++ calls Zig C ABI functions directly — no `#include "app_core.hpp"`, no `setBridge()`, no `shared_ptr`. See `zig-services/jni/README.md` for the C ABI interface documentation.

## Debugging

```bash
# Build .so with debug symbols
cd zig-services && zig build -Dtarget=aarch64-linux-android

# Run on device
cd .. && ./gradlew :app:installDebug

# View logs
adb logcat -s SDL:* Nexus:* Python:*
```

### Common issues

| Issue                        | Likely cause                           | Fix                                        |
|-----------------------------|----------------------------------------|-------------------------------------------|
| `UnsatisfiedLinkError`       | Zig `.so` not built or wrong target    | Rebuild with correct `-Dtarget`            |
| `Python startup failed`      | Chaquopy not initialized before bridge | Move `installPythonBridge` before SDL init |
| `ImGui touch not responding` | Missing SDL touch events               | Enable `SDL_HINT_TOUCH_MOUSE_EVENTS`       |
| `Lua archive not found`      | `packLuaDat` not run                   | `./gradlew :app:packLuaDat`                |

## Themes

Default: `nexus-field` (high contrast, larger touch padding). Override in `nxs_config.json`:

```json
{ "theme": "nexus-dark" }
```

## Blueprint & flows

`blueprint.json` defines app structure (same schema as desktop). `flows/flows.json` adds runtime automations. Enable in `nxs_config.json`:

```json
{ "flows": { "enabled": true } }
```

Schema: [blueprint-schema.md](../../docs/templates/blueprint-schema.md)

## Cross-language bridge

Same architecture as desktop — C++, Python (Chaquopy), Lua (sol2), and Zig JNI call each other in-process through the `AppCore` bridge.
